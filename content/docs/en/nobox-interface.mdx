In general, NoBox.AI has four main types of views: Dashboard, Kanban, List, and Form. these four views are integrated to support the user's workflow from start to finish.

---

## **Dashboard**

The Dashboard is the main page that displays a summary of activities and important information in real-time. This page is usually the first view that appears after a user successfully logs into the system.

Through the dashboard, users can monitor service performance, data development, operational trends, and overall system conditions in a single view.

Information on the dashboard is presented in the form of charts, statistics, and visual summaries, making it easier for users to understand data without having to open other menus.

This view is very helpful in the decision-making process as it presents data that has been processed automatically by the system.

### Main Functions of the Dashboard:

- Displays a summary of system performance.
- Provides brief reports on user activities.
- Monitors service status in real-time.
- Displays important notifications.
- Supports evaluation and decision-making.

### Common Features on the Dashboard:

| Feature / Button | Function |
| :--- | :--- |
| **Year Filter** | Displays data based on a specific time period. |
| **Auto Refresh** | Automatically updates data every 10 seconds. |
| **Help** | Displays usage guides and the knowledge base. |
| **Full Screen** | Displays charts in full-screen mode. |

---

## **Kanban**

The Kanban view is used to manage workflow and business processes visually. In this view, data is organized in boards, columns, and cards.

Each card usually represents a single activity, task, or transaction being processed by the system.

With this view, users can easily know the status of work, move data between stages, and monitor overall progress.

The Kanban method is very effective for managing sales pipelines, offering processes, and internal company workflows.

### Main Functions of Kanban:

- Manages work processes step-by-step.
- Monitors the progress of each activity.
- Reduces the risk of work delays.
- Improves coordination between teams.
- Makes it easier to identify process bottlenecks.

### Common Features on Kanban:

| Feature / Button | Function |
| :--- | :--- |
| **Board** | Selects the project or data to be displayed. |
| **Filter** | Filters data based on specific criteria. |
| **Search** | Searches for data using keywords. |
| **Kanban** | Displays the workflow visually. |

---

## **List**

The List view presents data in a systematic table or list format. This view is very suitable for managing large amounts of data.

On the List page, users can see detailed information for each data entry more comprehensively in rows and columns.

Additionally, this view supports searching, sorting, and direct data processing.

Users can manage data such as adding, editing, deleting, or exporting data through this page.

### Common Features on List:

| Feature / Button | Function |
| :--- | :--- |
| **Search** | Searches for data based on keywords. |
| **Refresh** | Updates data manually. |
| **Column Picker** | Adjusts the column display on the table. |
| **Restore Default** | Returns column settings to their initial state. |
| **Export** | Downloads data in Excel or PDF format. |
| **Auto Refresh** | Automatically updates data every 10 seconds. |
| **Status Filter** | Filters data based on status or category. |

## **Form**

The Form view is used to add new data, edit existing data, and validate inputs before data is saved into the system.

Forms are designed with a clear structure so that users can fill in data easily and correctly.

Each field is usually tailored to system requirements and equipped with automatic validation.

This aims to minimize input errors and maintain data quality.

- Performs automatic validation.
- Maintains data consistency.

### Common Features on Forms:

| Feature / Button | Function |
| :--- | :--- |
| **Group / Category** | Groups fields for a neater and more structured look. |
| **Save** | Saves new data or changes to existing data. |
| **Delete** | Removes selected data from the system. |

## **Usage Tips**

To maximize the use of the NoBox.AI interface, consider the following tips:

- Use filters and search to speed up the data lookup process.
- Enable auto refresh when monitoring real-time data.
- Utilize the export feature for reporting needs.
- Adjust column displays according to your specific task requirements.
- Double-check data before saving through forms.
- Use the Kanban view to monitor work progress.
- Utilize the dashboard as your monitoring center.

</Tips>

## **Troubleshooting**

If you experience issues while using the NoBox.AI interface, follow these steps:

- Refresh the system page if data does not appear perfectly.
- Ensure your internet connection is stable and smooth.
- Re-check currently active filters and search columns.
- Clear your browser cache if some UI elements do not appear.
- Log out and log back in if system features are not responding.
- Contact the support team if technical issues persist.
