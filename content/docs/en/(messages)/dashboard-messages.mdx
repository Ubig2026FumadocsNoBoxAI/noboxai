## **Detailed Component Breakdown**

- **New**  
  This section displays the number of conversations not yet handled by agents. This information is used to track the volume of messages still awaiting a response.

- **In Progress**  
  Displays the number of conversations currently being handled by agents. This data shows the active workload of the team.

- **Completed**  
  Displays the number of conversations that have been finalized. This information helps evaluate the service resolution rate.

- **Archive**  
  Displays conversations that have been saved as archives. This data is used for service documentation.

- **Conversations Created**  
  Displays charts of message activity over time, helping to identify customer communication trends.

- **Agents & Supervisors**  
  Displays the number of active team members. This information is used to monitor resource availability.

- **Conversation Tags**  
  Displays message distribution according to tag categories. This feature simplifies classification and conversation analysis.

- **Agent Online Status**  
  Displays the online and offline status of agents, ensuring the service team's readiness.

- **Conversation Funnel**  
  Displays the journey of messages from entry to completion. This feature helps evaluate the effectiveness of service processes.

## **How to Read the Message Dashboard**

### Guidelines for Reading Data

To properly understand the data on the Message Dashboard, users should observe the following:

1. Monitor the number of new messages daily.
2. Compare the message volume with the number of active agents.
3. Track the status of "In Progress" vs "Completed."
4. Use charts to visualize communication trends.

### Data Interpretation

The dashboard data can be used to assess the quality of customer service. Supervisors or administrators can utilize this information as material for routine evaluations and team performance improvements.
