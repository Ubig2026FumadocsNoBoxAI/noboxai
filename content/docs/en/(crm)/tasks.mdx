This feature enables users to organize, monitor, and execute work activities through a structured environment. Whether managing personal reminders or complex team projects, the Tasks module ensures that workflows remain transparent, organized, and highly efficient.

## **Key Features and Buttons**

The following tools are available to help you manage your workflow effectively:

| Feature / Button  | Function                                                                    |
| ----------------- | --------------------------------------------------------------------------- |
| **Add Board**     | Creates a new project board or pipeline to group related tasks.             |
| **Add Task**      | Introduces a new individual action item into the current board.             |
| **Edit & Delete** | Provides management options to modify or remove pipeline board data.        |
| **Kanban View**   | Displays tasks in a visual, drag-and-drop board layout for easier tracking. |
| **Calendar View** | Visualizes tasks on a monthly/weekly calendar to manage deadlines visually. |
| **List View**     | Presents tasks in a systematic, sortable table format for deep data review. |

## **Managing Task Boards**

1. To create a new board, click the [ <img src="/images/id/Tombol%20Papan.png" alt="Add Board" className="inline h-5 w-auto mx-1" /> ] button.
2. An **[Add Board]** dialog will appear. Complete the Name, Description, and Stage List for your board.

### Add Board Form Fields

| Field       | Description        | Example                                        |
| ----------- | ------------------ | ---------------------------------------------- |
| Name        | Title of the board | Development Team                               |
| Description | Summary of usage   | Workflow for the software development cycle    |
| List        | Workflow stages    | New, In Progress, Ready for Testing, Completed |

Once configured, click **[Save]** to initialize the new task board.

---

### **Editing and Deleting Boards**

To modify a board, click the [ <img src="/images/id/Edit%20Delete2.png" alt="Edit Delete" className="inline h-5 w-auto mx-1" /> ] icon, select **[Edit]**, and apply your changes.

To remove a board, select **[Delete]** from the same menu and confirm your action when prompted.

<Callout type="info" title="Important Information">
  Board deletion is permanent. Ensure that all critical tasks within the board
  have been moved or archived before proceeding with deletion.
</Callout>

## **Collaboration and Utility**

Comments are used to provide feedback, context, or status updates on specific tasks.

1. Click the [ <img src="/images/id/Comment.png" alt="Comment" className="inline h-5 w-auto mx-1" /> ] icon.

<Tips>
## **Usage Tips**

To master your task management workflow:

- **Descriptive Titles**: Use clear names so team members understand objectives at a glance.
- **Detailed Planning**: Fill out the Plans and Next Steps fields to minimize confusion and follow-on meetings.
- **Priority Alignment**: Use Priority levels to ensure the most critical business objectives are handled first.
- **Dynamic Tagging**: Use tags to quickly group related tasks across different project boards.
- **Real-time Updates**: Move tasks to new stages as soon as progress is made to keep the pipeline accurate.
- **Centralized Chat**: Use the Comments section instead of external apps to keep task-related discussions in context.

</Tips>

## **Troubleshooting**

If you encounter technical difficulties with the Tasks module:

- Refresh the page if the dashboard or board views fail to render correctly.
- Verify that your internet connection is stable for real-time synchronization.
- Double-check your data input (ensure all required fields are filled).
- Confirm your account role has the necessary permissions for the specific board.
- Log out and log back in if the system feels unresponsive.
- Clear browser cache if system performance seems unusually slow.
- Access the platform via a different browser if display issues persist.
- Contact the support team if specific tasks are not saving or updating properly.

</Kendala>
