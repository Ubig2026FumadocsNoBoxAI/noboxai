**Form Tags** is a flexible organizational module in NoBox.AI used to classify your form data into identifiable groups.

This feature enables users to organize, filter, and search for specific forms or results based on custom labels, ensuring that your data management remains structured, intuitive, and highly efficient.

## **Adding a Form Tag**

To create a new organizational label, navigate to the **[Form Tags]** menu ➔ **[Add Form Tag]**.

An **[Add Form Tag]** dialog will appear.

<ImageZoom
  src="/images/id/Tambah%20Tag%20Formulir.png"
  alt="New Tag Entry Dialog"
  width={800}
  height={450}
  className="rounded-lg border shadow-sm mb-4"
/>

### **Steps to Add a Tag**

1. Click the **[Add Form Tag]** button.
2. Enter the **Tag Name** that corresponds to your desired category (e.g., "VIP," "Promo-2024," "Wholesale").
3. Click **[Save]** to commit the tag to the system.

You can add multiple tags sequentially to build a robust classification system.

## **Managing Tag Records**

### Editing Form Tags

To rename or update a tag, select the target record from the dashboard list. Modify the title and click **[Save]**. Changes are updated across all forms currently carrying that tag immediately.

### Deleting Form Tags

To remove a tag entirely, select it and click **[Delete]**. Verify the action in the confirmation prompt.

Click **[Yes]** to confirm or **[No]** to cancel.

<Callout type="info" title="Important Information">
Tag deletion is permanent. Once a tag is deleted, it will be automatically removed from all form templates and result records where it was previously applied.
</Callout>

## **Practical Application of Form Tags**

Tags are applied in two primary locations: the **[Form Results]** (submissions) and **[Form Templates]** (designs).

### **Applying Tags to Form Submissions (Results)**

1. Navigate to the **[Form Results]** menu.
2. Open a specific result record.
3. Click the **Tags** input field. A dropdown of your previously created tags will appear.

<ImageZoom

4. Select one or more relevant tags and click **[Save]**.

### **Applying Tags to Form Templates**

1. Navigate to the **[Form Templates]** menu.
2. Edit an existing template or click "Add New Template."
3. Locate the **Tag** field. Select the appropriate tags from the list.

4. Click **[Save]** to finalize the template configuration.

## **Advanced Filtering with Tags**

The primary benefit of tagging is the ability to isolate specific data segments rapidly.

### **How to Filter by Tag**

1. Open either the **[Form Results]** or **[Form Templates]** dashboard.
2. Click the **Tag Filter** button.
3. Select the specific tags you wish to view.
4. The system will refresh to show only the data carrying those selected labels.

<Tips>
## **Tagging Strategy Tips**

- **Naming Conventions**: Use consistent terminology (e.g., "Region-East" vs "EastRegion") to prevent fragmented data.
- **Relevant Context**: Use tags to indicate customer segments, campaign sources, or priority levels.
- **Multi-Tagging**: Don't hesitate to use multiple tags if a form falls into more than one category.
- **Audit**: Periodically review your tag list to delete obsolete or duplicate tags, keeping your dropdown menus clean.

</Tips>

<Kendala>
## **Troubleshooting**

If you experience issues with Form Tags:

- **Missing Tags**: If a tag doesn't appear in the dropdown, verify that it was successfully saved in the master [Form Tags] menu.
- **Creation Failures**: Check your internet stability or refresh the page if the "Add Tag" dialog fails to close after saving.
- **Filtered Data Inconsistency**: If a filter returns no results, ensure that the tags have actually been assigned to the individual records, not just created in the master list.

</Kendala>
