The **Subscription** (Berlangganan) menu is the central hub for cost information and service plans you use in NoBox.Ai. Through this page, you can monitor available quotas, track storage usage, and view the transaction history of products purchased.

This page is very important for business owners to ensure all system operations continue to run smoothly by routinely monitoring account capacity.

The interface above displays a summary of your subscription plan. This page is divided into several primary information modules:

| Component | Primary Function |
| :--- | :--- |
| **Members** | Displays total registered team members or agents. |
| **Accounts** | Displays the number of connected social media/marketplace accounts. |
| **Support** | Information regarding your support plan level. |
| **Storage** | Percentage of storage memory used for attachments and databases. |
| **Order History** | List of all transactions or invoices for NoBox product purchases. |
| **Quota** | Details of data extraction allowance and broadcast message limits. |

<Tips>
## **Usage Tips**

Some tips for managing your subscription efficiently:

- Check quotas regularly, especially before running large promotional campaigns.
- Monitor the storage bar; if it is approaching the limit, perform old file cleanup or upgrade your plan.
- Use Order History as a resource for your company's internal financial reporting.
- Ensure the Support level matches your business needs to receive priority responses when issues occur.

</Tips>

<Kendala>
## **Troubleshooting**

If you encounter discrepancies in subscription information, consider the following steps:

- **Quota Not Increasing After Top-up**: Refresh the page or wait 5-10 minutes for the system synchronization to complete.
- **Storage Full**: Delete some large file attachments in File Manager or contact us to increase capacity.
- **Order History Empty**: Ensure you are logged in with the same account used when making the purchase.
- **Data Not Appearing**: Check your internet connection or ensure no ad-blockers are interfering with the system display.

</Kendala>
