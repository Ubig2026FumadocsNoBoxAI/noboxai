Custom Fields is a powerful extensibility module in NoBox.AI that allows users to append supplementary data columns to the system's default data structures, tailored specifically to your unique business logic.

This menu enables you to customize contact and product data architectures, ensuring your database contains every piece of information relevant to your operations. By utilizing custom fields, specialized data points unique to your industry can be captured, stored, and managed natively within the NoBox.AI platform.

## **Creating a New Custom Field**

1. Navigate to the **[Custom Fields]** menu ➔ select either the **[Contact]** or **[Product]** category depending on where the data belongs.
2. Click **[Add Custom Field]** to launch the configuration dialog.

### **Configuration Field Definitions**

Complete all technical parameters within the dialog:

| Configuration | Description                                                                                                                                         | Example          |
| ------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------- |
| Input Element | The type of UI element used in the form (e.g., standard text input, dropdown menu, checkbox).                                                       | Input            |
| Input Type    | The technical data type accepted by the column (Text, Number, Email, Date, etc.). Selecting the correct type ensures data validation and integrity. | Number           |
| Required      | A toggle defining if the field must be completed before saving a record. Enable this for critical business data (e.g., Secondary Phone).            | -                |
| Label         | The title of the field displayed on the front-end form. Use short, descriptive names.                                                               | Alternate Number |
| Placeholder   | A subtle hint or example value displayed inside the empty field to guide the user (e.g., "62xxxxxx").                                               | 62xxxxxx         |
| Default Value | A pre-filled value that automatically appears in the field for every new entry, saving time for repetitive data points.                             | -                |

Click **[Save]** to commit the new attribute to your data structure.

## **Managing Field Records**

### **Editing Custom Fields**

To modify a label or placeholder, select the target record and apply changes in the dialog. Click **[Save]**. These updates will propagate immediately across all forms utilizing that specific field.

### **Deleting Custom Fields**

Select the obsolete field and click **[Delete]** ➔ confirm in the prompt.

<Callout type="info" title="Important Information">
  Deleting a custom field removes it from all existing contacts or products.
  This action also purges any data previously stored in that column. Ensure the
  historical data is no longer necessary before finalizing the deletion.
</Callout>

---

## **Implementation in Forms**

Once successfully configured, custom fields automatically appear at the bottom of the relevant Contact or Product forms.

1. Navigate to the **[Contacts]** or **[Products]** module ➔ Open an existing record or create a new one.
2. Scroll to the bottom of the standard form. Your newly created custom fields will be rendered automatically.
3. Enter the required data ➔ Click **[Save]** to update the record with its integrated additional information.

<Tips>
## **Workflow Best Practices**

- **Descriptive Labels**: Use internal naming conventions that are easily understood by the entire sales and support team.
- **Strategic Validation**: Use the **[Required]** toggle for data points essential for shipping or billing to prevent incomplete records.
- **Type Sensitivity**: Always use the correct Input Type (e.g., 'Date' for birthdays, 'Number' for price aliases) to ensure your data stays "clean."
- **Logical Ordering**: Arrange your custom fields in a sequence that follows your team's natural data entry habits to maximize efficiency.
- **Guided Entry**: Utilize Placeholders to show users the exact data format expected (e.g., "DD-MM-YYYY").

</Tips>

<Kendala>
## **Troubleshooting**

- **Visibility Delay**: If a new field doesn't appear in the form immediately after saving, refresh your browser page to sync the layout.
- **Validation Errors**: If the system rejects a save, check if the data entered in a custom field matches its assigned **Input Type** (e.g., trying to save text in a 'Number' field).
- **Access Restrictions**: If you cannot create or remove fields, verify with your administrator that your role has the necessary "Settings Management" permissions.
- **Session Expiry**: Log out and log back in if the Custom Fields menu fails to load or save after an extended period of inactivity.
- **Caching issues**: Clear browser cache if labels or placeholders appear outdated after an edit.
- **Schema Conflicts**: If deleting a field causes database errors, contact support immediately to verify your organization's data schema integrity.

</Kendala>
